üìå 1. The Core Problem (Masla Kya Hai?)

Distributed systems mein computers alag alag hotay hain (Heterogeneity).

  • Data Representation:
    • Big-Endian: MSB (Most Significant Byte) pehle aata hai. (Insani tareeqa).
    • Little-Endian: LSB (Least Significant Byte) pehle aata hai. (Ulta tareeqa).
    • Risk: Agar 0x1234 bheja aur receiver ne ulta parha, to value change ho jayegi.
  • Character Coding:
    • ASCII: 1 Byte per character (English only).
    • Unicode: 2 Bytes per character (Multi-language).

‚öôÔ∏è 2. The Process (Kaam Kaise Hota Hai?)

Objects ko network par bhejne ka tareeqa:

  1. Flattening: Data structures ko tod kar bytes ki line banana.
  2. Marshalling: Data ko agreed format mein pack karna (Encoding).
  3. Transmission: Message network par jata hai.
  4. Unmarshalling: Bytes ko wapas jod kar Data/Object banana (Decoding).

üßÆ 4. CORBA CDR Rules (Most Important for Numericals)

Agar exam mein calculation aaye, to ye rules yaad rakhein:

  • Primitive Types:
    • Char: 1 Byte (Kahin bhi aa sakta hai).
    • Short: 2 Bytes (Start index Even hona chahiye: 0, 2, 4...).
    • Long / Float / Enum: 4 Bytes (Start index Multiple of 4 hona chahiye: 0, 4, 8...).
    • Double: 8 Bytes (Start index Multiple of 8 hona chahiye).
  • Strings:
    • Pehle Length (Unsigned Long ‚Äì 4 Bytes) aati hai ‚Üí Phir Characters.
    • Note: Length hamesha 4-byte boundary se shuru hogi.
  • Padding:
    • Agar agla data apni sahi jagah (Alignment) par nahi aa raha, to beech mein Zeroes (Padding) bharnay parengay.
