Topic: Classification of Computer Architectures
Proposed by: Michael Flynn (1966)
Basis: Yeh classification 2 cheezon par base karti hai:

  1. Instruction Stream: (Command kya hai?)
  2. Data Stream: (Kis data par kaam ho raha hai?)

1. SISD (Single Instruction, Single Data)

  • Definition: Yeh architecture ek Processor use karta hai jo ek waqt mein ek Instruction ko ek hi Data Stream par execute karta hai.
  • Working Mechanism:
    • Instructions line-by-line (sequential) chalti hain.
    • Parallel processing nahi hoti.
    • Data fetch aur execute hone ka process serial hota hai.
  • Key Characteristics:
    • Simplest architecture hai.
    • Speed limited hoti hai (Internal transfer rate par depend karti hai).
    • Isay "Sequential Computer" bhi kehte hain.
  • Examples: Traditional PCs, Old Mainframes, Single-core workstations.

2. SIMD (Single Instruction, Multiple Data)

  • Definition: Yeh architecture Multiple Processors use karta hai jo sab Same Instruction follow karte hain lekin Alag Alag Data par kaam karte hain.
  • Working Mechanism:
    • Control Unit ek command bhejta hai (e.g., "Multiply by 2").
    • Sab processors ek hi waqt mein (simultaneously) apne apne data hissay par ye command chalate hain.
    • Yeh Vector/Matrix operations ke liye best hai.
  • Key Characteristics:
    • Lockstep Operation: Sab processors sync mein kaam karte hain.
    • Categories (1972 Paper):
      1. Array Processor: Har processor ki apni alag memory hoti hai.
      2. Pipelined Processor: Instruction same hoti hai, data central resource se tukron (fragments) mein process hota hai.
      3. Associative Processor: Har processor apne local data ke hisaab se faisla karta hai ke instruction man'ni hai ya skip karni hai (Predicated Execution).
  • Examples: GPUs (Graphics Cards), Cray Supercomputers, Scientific Modeling.

3. MISD (Multiple Instruction, Single Data)

  • Definition: Yeh architecture Multiple Processors use karta hai jo Alag Alag Instructions execute karte hain lekin Same Data Stream par.
  • Working Mechanism:
    • Ek hi data multiple processors ke paas jata hai.
    • Har processor us data par different calculation karta hai.
  • Key Characteristics:
    • Impractical: General computing mein iska koi khas use nahi hai.
    • Fault Tolerance: Yeh sirf wahan use hota hai jahan checking karni ho (Redundancy).
    • Commercially available machines nahi miltin.
  • Examples: Space Shuttle Flight Control Systems.

4. MIMD (Multiple Instruction, Multiple Data)

  • Definition: Yeh architecture Multiple Processors use karta hai jo Alag Alag Instructions aur Alag Alag Data par kaam karte hain.
  • Working Mechanism:
    • Har processor independent hota hai.
    • Processors Asynchronously kaam karte hain.
    • Yeh true parallel computing hai.
  • Key Characteristics:
    • Memory Models:
      1. Shared Memory: Sab processors global memory use karte hain.
      2. Distributed Memory: Har processor ki local memory hoti hai.
    • Sabse zyada flexible aur powerful model hai.
  • Examples: Modern Multi-core PCs, Clusters, Supercomputers.

5. Execution Models (MIMD Programming Styles)

(Yeh hardware nahi, software execution ka tareeqa hai)

A. SPMD (Single Program, Multiple Data)

  • Definition: Sab processors par Code same hota hai, Data alag.
  • Logic: Processors apni marzi se code ke alag parts run kar sakte hain (Conditional paths).
  • Use: AI, Deep Learning training.

B. MPMD (Multiple Program, Multiple Data)

  • Definition: Sab processors par Code bhi alag hota hai, Data bhi alag.
  • Logic: Manager-Worker approach.
  • Use: Heterogeneous systems.
