Lecture Overview: Challenges in Distributed Systems

Distributed Systems ka matlab hai jab bohot saray computers mil kar ek single system ki tarah kaam karein. Lekin jab computers alag alag jagah hotay hain aur alag alag types ke hotay hain, toh bohot saray maslay (challenges) aatay hain.

Yeh lecture mainly 4 Challenges ke baaray mein hai:

  1. Heterogeneity (Variety/Difference)
  2. Openness (Extendibility)
  3. Security (Hifazat)
  4. Scalability (Growth handle karna)

1. Heterogeneity (Mukhtalif Qisam ki Cheezein)

Concept:
"Heterogeneity" ka simple matlab hai "variety" ya "different types". Distributed systems mein har cheez same nahi hoti.

  • Networks: Koi WiFi use kar raha hai, koi Ethernet (LAN cable).
  • Hardware: Kisi ke paas Intel processor hai, kisi ke paas Mac ka M1 chip. Data store karne ka tareeqa hardware ke hisaab se badal jata hai.
  • Operating Systems: Windows, Linux, Mac — sabki system calls different hoti hain.
  • Programming Languages: Java, C++, Python sab data ko alag tarah se handle karte hain.

Problem:
Agar mera code Python mein Windows par chal raha hai, aur aapka code C++ mein Linux par, toh yeh aapas mein baat (communicate) kaise karenge?

Solution (Middleware & Virtual Machines):

  • Protocols: Internet ne yeh masla solve kiya. Sab IP protocol use karte hain.
  • Middleware: Yeh differences ko mask kar deta hai, translator ki tarah kaam karta hai.
  • Mobile Code & Virtual Machines (VM): Java ka code JVM par run hota hai, isliye har jagah chal jata hai.

2. Openness (Khulaa System)

Concept:
Open system wo hota hai jisme naye features add karna aasaan ho bina purana system toray.

Key Requirement:
Openness ke liye zaroori hai ke system ke Interfaces published hon.

  • Developers ko pata hona chahiye ke system se connect kaise karna hai.
  • Iske liye IDL (Interface Definition Language) use hoti hai — jo syntax batati hai.
  • Challenge: IDL semantics nahi batati — yeh documentation se samajhna hota hai.

3. Security (Hifazat)

Concept:
Distributed systems mein data network par travel karta hai, isliye security bohot critical hai. Security ke 3 pillars hain:

  • Confidentiality: Privacy.
  • Integrity: Data change na ho.
  • Availability: Server down na ho.

Solution:

  • Encryption: Data ko readable se code mein convert karta hai.
  • Authentication: Passwords / OTPs / Biometrics se identity confirm hoti hai.

4. Scalability (Phelne ki Salaahiyat)

Concept:
Agar users 10 se barh kar 10 million ho jayen, toh system sustain kare — isay Scalability kehte hain.

3 Dimensions:

  • Size: Ziada users add ho sakein.
  • Geographical: Dunya mein kahin bhi users hon — system slow na ho.
  • Administrative: Multiple organizations mein manage ho sake.

Problems:

  • Centralized server bottleneck.
  • Network latency — LAN fast, WAN slow.

5. Scaling Techniques (Solutions)

A. Hiding Communication Latencies

Asynchronous communication — request bhej kar wait nahi karte, CPU aur kaam karta rehta hai.

B. Distribution (Kaam Baantna)

DNS Example: Tree structure — load divide ho jata hai, searching fast ho jati hai.

C. Replication (Copy Banana)

Data ki copies multiple jagah rakhna — caching — load reduce hota hai.
